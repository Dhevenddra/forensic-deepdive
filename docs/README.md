# docs/

Reference documentation. Consult only when revisiting a decision or onboarding a deep contributor. **Not part of the runtime brain** — Claude Code does NOT read these on every session.

- **RESEARCH.md** — landscape research that produced the architecture. Tool comparisons, dying-tool callouts, threshold rationales.
- **ARCHITECTURE.md** — design rationale. The "why" behind tool choices, data flow, distribution decision, polyglot strategy, update semantics.
- **ROADMAP.md** — v0.2 and v1.0 plans with effort estimates.

If you (Claude Code or human contributor) find yourself wanting to override a DECISIONS.md entry, read the relevant section here first. The deeper context might change your mind, or sharpen the superseding entry.
